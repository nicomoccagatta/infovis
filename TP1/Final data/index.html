01 · Introducción

Introducción

Consigna del TP y enfoque del proyecto

Esta entrega cruza hábitos de escucha y actividad técnica para responder una pregunta: si el contexto musical acompaña de forma consistente mis momentos de mayor foco.

El TP pide contar una historia con datos personales en una página web, integrando visualizaciones estáticas y/o interactivas con las herramientas vistas en clase (RAWGraphs, Flourish, Datawrapper y Tableau).

Tomé dos historiales personales: Spotify y GitHub. En Spotify agregué por día el artista más escuchado y el tiempo escuchado. En GitHub conté las contribuciones por día. Al juntar ambas señales, busco visualizar si ciertos artistas se asocian con mis días de mayor productividad.

Lectura guía: la narrativa avanza desde contexto y metodología hacia evidencias visuales comparables entre herramientas.

02 · Contexto y armado del dataset

Desarrollo

Integración Spotify + GitHub por día

El dataset se construye con una lógica temporal común para hacer legible la relación entre escucha musical y productividad en desarrollo.

Tomé mi historial de Spotify y lo convertí en una serie temporal diaria.

Para cada día identifiqué el artista más escuchado y el tiempo total de escucha.

En paralelo, usé mi historial de GitHub para construir otra serie diaria: contribuciones por día. El proceso de armado del dataset y el repositorio quedan documentados aquí:

GitHub: nicomoccagatta/infovis

Luego uní ambas señales por fecha para leer relaciones entre hábitos y productividad. La pregunta guía es: ¿qué artista aparece más asociado a mis jornadas de mayor actividad?

Con esa base armé las 4 visualizaciones integradas (RAWGraphs, Flourish, Datawrapper y Tableau), cada una formulada como una pregunta que orienta la interpretación y sustenta las conclusiones.

Qué cambia a partir de acá: cada sección visual presenta una hipótesis de lectura y un foco interpretativo distinto.

Con el dataset armado, empieza la exploración visual. Cada herramienta ofrece una lente distinta para leer la misma pregunta.

07 · Hallazgos principales

Lo que revelan los datos

Patrones clave detectados a lo largo del análisis

Antes de cerrar, estas son las señales más claras que surgieron al cruzar ambas fuentes de datos.

Hallazgo 01

La productividad en código se concentra en ciertos artistas y géneros recurrentes, no en una distribución homogénea.

Hallazgo 02

Los días de mayor actividad en GitHub coinciden con sesiones de escucha más largas e ininterrumpidas.

Hallazgo 03

La semana laboral muestra ritmos más marcados que el promedio diario, con picos claros de alineación música-código.

Hallazgo 04

No todos los hábitos de escucha acompañan la misma actividad: hay clústers con comportamientos diferenciados según el tipo de tarea.

08 · Reflexión final

Cierre y conclusiones

Spotify + GitHub como narrativa integrada de productividad personal

La música no solo acompaña el trabajo: marca su ritmo.

A lo largo de este análisis, los datos muestran que la relación entre hábitos de escucha y productividad no es casual ni lineal. Aparecen patrones consistentes: ciertos géneros y artistas se asocian con sesiones de mayor foco, y los ritmos semanales revelan una estructura más predecible de lo esperado.

Cruzar Spotify con GitHub fue una forma de convertir dos historiales personales en una narrativa visual sobre cómo trabajo, cuándo me concentro y qué contexto sonoro acompaña mis mejores momentos de desarrollo.

"No elegimos la música que escuchamos al trabajar por azar. Esa elección dice algo sobre cómo procesamos, cómo nos concentramos y cómo producimos."

Este proyecto deja una base concreta para seguir explorando: ¿qué pasa si sumamos más señales — horas del día, tipo de tarea, duración de las sesiones? La pregunta queda abierta, y los datos siguen ahí.
